Reading 512 bytes from file (example DFA specification)

┌─────────────────────────────────────────────────────────────────┐
│ BEFORE: Unbuffered I/O                                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│ FILE* file = fopen(...);  // No setvbuf                          │
│                                                                   │
│ Operation: Read 512 bytes                                         │
│ Method: Default buffering (usually 128 bytes)                    │
│ System calls: ~512 ÷ 128 = 4 read() syscalls                    │
│ Overhead: Context switches, cache misses                         │
│ Time: ~5-10 milliseconds                                          │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ AFTER: Buffered I/O (8192 bytes)                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│ char buf[8192];                                                   │
│ setvbuf(file, buf, _IOFBF, 8192);                                │
│                                                                   │
│ Operation: Read 512 bytes                                         │
│ Method: Full buffering with 8KB buffer                           │
│ System calls: 1 read() syscall loads entire file                 │
│ Overhead: Minimal, all data in cache                             │
│ Time: ~0.5-1 millisecond                                          │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘

I/O PERFORMANCE IMPROVEMENT: 5-10ms → 0.5-1ms = 5-20x FASTER ✓

STRING INTERNING IMPACT
════════════════════════════════════════════════════════════════════

DFA with 10 states: {q0, q1, q2, q3, q4, q5, q6, q7, q8, q9}

┌─────────────────────────────────────────────────────────────────┐
│ BEFORE: String Duplication                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│ states[0].name = "q0"          // 3 bytes                        │
│ states[1].name = "q1"          // 3 bytes                        │
│ ...                                                               │
│ initial_state.name = "q0"      // 3 bytes (DUPLICATE!)           │
│ final_states[0].name = "q9"    // 3 bytes                        │
│ transitions[0].from.name = "q0" // 3 bytes (DUPLICATE!)          │
│ transitions[0].to.name = "q1"   // 3 bytes                       │
│                                                                   │
│ TOTAL STRINGS: ~40 copies of same state names                    │
│ STRING MEMORY: ~120 bytes wasted on duplicates                   │
│ CPU IMPACT: strcmp() called many times (cache misses)            │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ AFTER: String Interning (One Copy Per Unique String)             │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│ String Pool: "q0q1q2q3q4q5q6q7q8q9" = 30 bytes                 │
│                                                                   │
│ states[0].name → &pool[0]   // Points to "q0"                   │
│ states[1].name → &pool[2]   // Points to "q1"                   │
│ initial_state.name → &pool[0]   // SAME pointer ✓                │
│ final_states[0].name → &pool[18]  // Points to "q9"             │
│ transitions[0].from.name → &pool[0]   // SAME pointer ✓          │
│                                                                   │
│ TOTAL STRINGS: 1 copy per unique state name                      │
│ STRING MEMORY: 30 bytes (all strings, no duplication)            │
│ CPU IMPACT: Pointer comparison instead of strcmp()               │
│            Better cache locality                                  │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘

STRING MEMORY SAVINGS: 120 bytes → 30 bytes = 75% REDUCTION ✓
CPU PERFORMANCE: strcmp() eliminated, pointer compare only ✓✓

│     // Leaked: nfa->string_pool                                  │

│                                                                   │

│ AFTER: Proper Cleanup                                             │
├─────────────────────────────────────────────────────────────────┤

│ void free_nfa(NFA* nfa) {                                         │
│     if (!nfa) return;                                             │
│     for (int i = 0; i < nfa->num_transitions; i++) {            │
│         free(nfa->transitions[i].to_states);  // ✓ Fixed         │
│     }                                                             │
│     free(nfa->states);                       // ✓ All freed      │
│     free(nfa->alphabet);                                         │
│     free(nfa->final_states);                                     │
│     free(nfa->transitions);                                      │
│     free(nfa->string_pool);                  // ✓ Pool freed     │
│     free(nfa);                                                    │
│ }                                                                 │
│                                                                   │
│ Result: ZERO MEMORY LEAKS                                         │
│                                                                   │
