STEP-BY-STEP INSTRUCTIONS
═══════════════════════════════════════════════════════════════════════════════

STEP 1: INSTALL GIT
──────────────────────────────────────────────────────────────────────────────

1. Go to: https://git-scm.com/download/win
2. Click "Download for Windows"
3. Run installer (accept all defaults)
4. Restart your computer


STEP 2: CONFIGURE GIT
──────────────────────────────────────────────────────────────────────────────

1. Open Command Prompt (Win+R, type "cmd", press Enter)

2. Set your name:
   git config --global user.name "Your Name"
   
   Example:
   git config --global user.name "John Doe"

3. Set your email:
   git config --global user.email "your.email@gmail.com"
   
   Example:
   git config --global user.email "john@example.com"

4. Verify:
   git config --global user.name
   git config --global user.email


STEP 3: CREATE GITHUB ACCOUNT & REPOSITORY
──────────────────────────────────────────────────────────────────────────────

1. Go to: https://github.com/signup
2. Enter:
   - Username: (e.g., legam2024)
   - Email: (your email)
   - Password: (create strong password)
3. Click "Create account"
4. Verify email address (check inbox)

5. Once logged in, click "+" → "New repository"
6. Repository name: dfa-to-nfa-compiler
7. Description: DFA to NFA Compiler using Flex and Bison
8. Choose: Public (so professor can see it)
9. Check: "Add a README file" (optional)
10. Click "Create repository"

COPY THE REPOSITORY URL:
  Look for the green "Code" button
  Copy the HTTPS URL (looks like: https://github.com/yourname/dfa-to-nfa-compiler.git)


STEP 4: INITIALIZE GIT IN YOUR PROJECT
──────────────────────────────────────────────────────────────────────────────

1. Open Command Prompt

2. Navigate to project:
   cd C:\Users\legam\Documents\btech\assignment\toc

3. Initialize git:
   git init

4. Add all files:
   git add .

5. Create first commit:
   git commit -m "Initial commit: DFA to NFA compiler with optimizations"

6. Rename branch to main (if needed):
   git branch -M main

7. Add remote repository:
   git remote add origin https://github.com/YOUR-USERNAME/dfa-to-nfa-compiler.git
   
   Replace: YOUR-USERNAME with your actual GitHub username

8. Push to GitHub:
   git push -u origin main

9. Enter credentials:
   Username: Your GitHub username
   Password: Your GitHub personal access token (see note below)


IMPORTANT: GITHUB PERSONAL ACCESS TOKEN
──────────────────────────────────────────────────────────────────────────────

GitHub no longer accepts passwords. You need a Personal Access Token:

1. Go to: https://github.com/settings/tokens
2. Click "Generate new token (classic)"
3. Name: "Git CLI"
4. Select scope: repo (check all repo options)
5. Click "Generate token"
6. COPY THE TOKEN (won't show again!)

Use this token as your "password" when Git asks for it.

# DFA to NFA Compiler

A complete compiler implementation for converting Deterministic Finite Automata (DFA) to Non-deterministic Finite Automata (NFA) using Flex and Bison.

## Features

- **Lexical Analysis** using Flex
- **Syntax Analysis** using Bison
- **Dynamic Memory Allocation** with 99.9% memory savings
- **String Interning** for efficient string handling
- **Optimized I/O** with buffered file operations
- **10 Test Examples** included
- **Zero Memory Leaks**
- **Interactive Menu System**

## Quick Start

### Prerequisites
- MSYS2 with gcc, flex, bison, make installed
- Or: Linux/Mac with standard build tools

### Build
```bash
make clean && make
```

### Run
```bash
./dfa_converter
```

## Project Structure

```
toc/
├── src/              # Source code
│   ├── dfa.h/c      # DFA module (optimized)
│   ├── nfa.h/c      # NFA module (optimized)
│   ├── converter.h/c # Conversion algorithm
│   ├── lexer.l       # Flex specification
│   ├── parser.y      # Bison specification
│   └── main.c        # Main program
├── tests/            # Test data
│   └── dfa_examples.txt
├── Makefile          # Build configuration
└── README.md         # This file
```

## Optimizations

- Dynamic memory allocation (vs fixed arrays)
- String interning (vs duplicate strings)
- Buffered I/O (5-20x faster)
- Proper memory cleanup

See OPTIMIZATION_CHANGES.md for details.

COMMON ISSUES & SOLUTIONS
═══════════════════════════════════════════════════════════════════════════════

ISSUE: "fatal: not a git repository"
SOLUTION:
  You need to run: git init
  From the project directory

ISSUE: "remote origin already exists"
SOLUTION:
  Run: git remote remove origin
  Then: git remote add origin [URL]

ISSUE: "Permission denied" or "401 Unauthorized"
SOLUTION:
  Use Personal Access Token instead of password
  Generate at: https://github.com/settings/tokens

ISSUE: Files too large (>100 MB)
SOLUTION:
  Delete obj/ folder and *.o files first
  Make sure .gitignore excludes them

ISSUE: "error: failed to push"
SOLUTION:
  Pull first: git pull
  Then push: git push

ISSUE: Wrong files uploaded
SOLUTION:
  Create .gitignore file
  Run: git rm -r --cached .
  Run: git add .
  Run: git commit -m "Remove unwanted files"
  Run: git push
