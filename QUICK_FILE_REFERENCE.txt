CORE PROJECT FILES (9 ESSENTIAL SOURCE FILES)
═══════════════════════════════════════════════════════════════════════════════

1️⃣  dfa.h (1 KB)
    What: DFA data structure definitions
    Contains: typedef struct DFA { ... };
    Provides: DFA* create_dfa(), void add_state(), void add_symbol(), etc.
    Why: Defines what a DFA is in memory

2️⃣  dfa.c (7 KB)
    What: Implementation of DFA functions
    Implements: Memory allocation, dynamic arrays, string interning
    Optimization: Dynamic allocation, string pooling
    Why: All DFA logic is here

3️⃣  nfa.h (1.5 KB)
    What: NFA data structure definitions
    Contains: typedef struct NFA { ... };
    Provides: NFA* create_nfa(), void add_nfa_state(), etc.
    Why: Defines what an NFA is in memory

4️⃣  nfa.c (7 KB)
    What: Implementation of NFA functions
    Implements: Multi-target transitions, proper cleanup
    Optimization: Dynamic arrays, string pooling, proper free()
    Why: All NFA logic is here

5️⃣  lexer.l (3 KB)
    What: Flex specification for tokenizer
    Purpose: Breaks text input into tokens
    Example: "q0" → IDENT token, "a" → SYMBOL token
    Why: First stage of compilation (Lexical Analysis)

6️⃣  parser.y (2 KB)
    What: Bison specification for parser
    Purpose: Parses tokens according to grammar
    Calls: add_state(), add_symbol() as it parses
    Why: Second stage of compilation (Syntax Analysis)

7️⃣  converter.h (0.5 KB)
    What: Header for DFA to NFA converter
    Provides: NFA* convert_dfa_to_nfa(DFA* dfa);
    Why: Interface to conversion function

8️⃣  converter.c (1 KB)
    What: DFA to NFA conversion algorithm
    Logic: Copy states, symbols, transitions to NFA
    Purpose: Convert DFA representation to NFA representation
    Why: Core algorithm

9️⃣  main.c (5 KB)
    What: Main program with user interface
    Features:
      - Load 10 examples from file
      - Display interactive menu
      - Parse selected DFA
      - Convert to NFA
      - Display results
    Why: User interacts with this

═══════════════════════════════════════════════════════════════════════════════
FILES BY PURPOSE
═══════════════════════════════════════════════════════════════════════════════

COMPILATION PHASES:

Phase 1 - Lexical Analysis:
  Source: lexer.l (Flex specification)
  Generated: lex.yy.c (Tokenizer code)
  Purpose: Break input into tokens

Phase 2 - Syntax Analysis:
  Source: parser.y (Bison specification)
  Generated: parser.tab.c, parser.tab.h (Parser code)
  Purpose: Parse tokens into DFA structure

Phase 3 - Semantic Analysis:
  Source: dfa.c (with add_state, add_symbol, etc.)
  Purpose: Build DFA in memory during parsing

Phase 4 - Conversion:
  Source: converter.c
  Purpose: Convert DFA to NFA


MODULE ORGANIZATION:

DFA Module:
  - dfa.h ..................... Data structure
  - dfa.c ..................... Implementation
  Purpose: Handle DFA objects

NFA Module:
  - nfa.h ..................... Data structure
  - nfa.c ..................... Implementation
  Purpose: Handle NFA objects

Conversion Module:
  - converter.h ............... Interface
  - converter.c ............... Algorithm
  Purpose: Convert between formats

Main Module:
  - main.c .................... User interface
  Purpose: Interactive program
