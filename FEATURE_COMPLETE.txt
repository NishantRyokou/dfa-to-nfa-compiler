INPUT NFA (Non-deterministic):
  States: q0, q1, q2, q3
  Initial: q0
  Finals: q3
  Transitions:
    q0 --(a)--> q0, q1    ← Goes to BOTH states!
    q1 --(b)--> q2
    q2 --(b)--> q3

OUTPUT DFA (Deterministic):
  States: {q0}, {q0,q1}, {q2}, {q3}    ← Combined!
  Initial: {q0}
  Finals: {q3}
  Transitions:
    {q0} --(a)--> {q0,q1}      ← Tracks both simultaneously
    {q0,q1} --(a)--> {q0,q1}
    {q0,q1} --(b)--> {q2}
    {q2} --(b)--> {q3}

KEY OBSERVATION:
  When NFA allows multiple next states, DFA creates a COMBINED state
  representing all possible states the NFA could be in. This is the
  essence of the subset construction algorithm!

                      ALGORITHM IMPLEMENTED
═══════════════════════════════════════════════════════════════════════════════

NAME: Subset Construction (Powerset Algorithm)

STEPS:
  1. Start with {q0} (initial NFA state)
  2. For each DFA state S:
     a. For each symbol σ in alphabet:
        - Find all NFA states reachable from any state in S on σ
        - Create new DFA state T = {all reachable states}
        - Add transition: S --σ--> T
        - If T is new, add to queue
  3. DFA final states = any containing NFA final state
  4. Return DFA

COMPLEXITY:
  Time:  O(2^n × |Σ|) where n = NFA states, |Σ| = alphabet size
  Space: O(2^n) worst case (all possible subsets)

WHY IT WORKS:
  DFA must be deterministic (one transition per state+symbol).
  When NFA has multiple choices, DFA tracks ALL possibilities
  simultaneously by grouping states into combined states.
